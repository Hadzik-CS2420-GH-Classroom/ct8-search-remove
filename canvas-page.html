👨‍💻 Code-Together 8: Search & Remove

🎯 Overview

An in-class code-together activity deepening students' understanding of singly linked lists through Farr's Ice Cream's pre-order queue. The simpler operations (destructor, push_front, push_back, pop_front, pop_back, print) are already implemented — students focus on contains and remove, both of which apply the same trailing pointer pattern seen in pop_back.

📖 Learning Objectives

By completing this activity, you will be able to:

  • Use the trailing pointer pattern (previous/current) to locate and unlink a node by value
  • Implement contains as a read-only traversal with a single pointer
  • Implement remove with three distinct cases: empty list, head match, and middle/tail match
  • Explain why contains() is marked const and what the compiler enforces
  • Recognize that contains and remove are both O(n) on an unsorted singly linked list

📝 What You’ll Do

Open each source file and complete the TODO comments. Each TODO tells you what to write, and the DISCUSSION comments explain the “why” behind the code. We’ll work through these together in class.

📚 Teaching Order

Work through the files in the order main.cpp calls them:

1. SinglyLinkedList.cpp — contains and remove (6 TODOs)

Section What You’ll Learn
contains — Order Lookup read-only traversal with single pointer; return true on match, false at nullptr; const qualifier enforced by compiler
remove — Head case no previous node exists so delegate to pop_front(); avoids duplicating temp-pointer logic
remove — Middle/tail case trailing pointer setup (same as pop_back); stop on data match instead of end; previous->next = current->next then delete

2. main.cpp — Pre-order Queue Scenario (6 TODOs)

Section What You’ll Learn
Part 1 — pop_back last customer cancels; review the O(n) trailing pointer traversal from the given code
Part 2 — contains look up order IDs; demonstrates read-only O(n) search with no shortcut for unsorted data
Part 2 — remove cancel orders at tail, middle, then head to show all three cases in sequence

🔧 Troubleshooting

CMake does not configure automatically

  1. Make sure you have Desktop development with C++ workload installed in Visual Studio
  2. Go to Project → CMake Settings and ensure the configuration is correct
  3. Try Project → Delete Cache and Reconfigure
  4. Verify that CMakeLists.txt is in the root of your cloned repository

Tests fail to build

  • Google Test is fetched automatically via CMake’s FetchContent — make sure you have an internet connection during the first build
  • Do not modify header files in the include/ folder

Running the Program

  1. In Visual Studio, select ct8-search-remove.exe as the startup item from the dropdown
  2. Press Ctrl+F5 to run without debugging
  3. You should see output from each section as you complete the TODOs
Tip: Look for the // ! DISCUSSION: comments in each file — these explain the “why” behind each concept. The // ? SEE DIAGRAM: comments point you to visual aids in the images/ folder.
